# C/C++ 编程规范（基础版）

## 1. 总则

- 新增代码必须通过 `clang-format` 与 `clang-tidy`。
- 提交前必须通过本地 Hook 检查。
- 合并前必须通过 CI 质量门禁。

## 2. 风格规范

- 统一使用仓库根目录下 `.clang-format`。
- 禁止手写与规则冲突的格式；以格式化工具结果为准。
- 推荐每次提交前先执行格式化。

## 3. 命名规范

命名规则由 `.clang-tidy` 中 `readability-identifier-naming` 强制执行，核心约定：

- 类/结构体/函数：`CamelCase`
- 变量/成员：`lower_case`
- 成员变量后缀：`_`
- 全局常量：`UPPER_CASE`

## 4. 质量规范

- `.clang-tidy` 中启用的警告视为错误（`WarningsAsErrors: '*'`）。
- 对于明确豁免的检查项，需在规则文件或评审记录中说明原因。

## 5. Git 门禁策略

- `pre-commit`：检查已暂存的 C/C++ 文件格式。
- CI（PR/Push）：执行全仓格式检查与静态检查。
- 不通过门禁的变更不得合并到主分支。

## 6. 例外管理

- 第三方代码、自动生成代码建议放在约定目录并在脚本中排除。
- 任何新增排除规则必须经过代码评审。
